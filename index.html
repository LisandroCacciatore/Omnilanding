El flujo del atleta

De datos crudos a decisión
en un solo flujo

01

Wellbeing Check — pre-sesión obligatorio

El alumno reporta sueño, energía y dolor antes de entrenar. El sistema calcula un score 0-100 y ajusta la carga sugerida si el contexto es adverso.

02

Sesión activa — card deck con carry-over

Interfaz optimizada para registrar cada set con peso y reps. El sistema recuerda el último peso, propaga ajustes a sets siguientes y detecta 2+ sets pesados para sugerir bajar carga.

03

Persistencia anti-fallback — dato garantizado

La sesión se guarda vía API con fallback directo a Supabase. Si ambos fallan, se muestra un error con botón de reintento. El dato nunca se pierde.

04

Vistas SQL — inteligencia en la DB

Cada log alimenta las 4 vistas analíticas. v_athlete_risk combina 5 CTEs para producir el score de riesgo del atleta en tiempo real.

05

Decisión asistida — donde está el valor

El profesor ve el semáforo de riesgo, la auto-progression sugerida por ejercicio, la predicción de sesión y el estancamiento detectado. Decide con datos, no con intuición.

Preguntas frecuentes

La mayoría de las herramientas resuelven la capa de gestión (rutinas, comunicación). TechFitness además calcula un score de rendimiento 0-100 por atleta, detecta riesgo de sobrecarga y abandono en tiempo real, y genera sugerencias de auto-progression basadas en datos históricos. La lógica vive en vistas SQL en la base de datos, no en el frontend — eso la hace consistente en cualquier dashboard o reporte.

El motor de programas tiene foco en fuerza (Starting Strength, Wendler, PPL, etc.) porque ahí está el dominio más profundo. Pero la gestión de alumnos, membresías, sesiones y analítica de rendimiento funciona para cualquier modalidad. Si trabajás con atletismo, funcional o deportes, la plataforma se adapta — los programas pre-armados son opcionales.

Acceso completo a TechFitness con tu gimnasio real. Podés cargar alumnos, asignar rutinas, ejecutar sesiones y ver cómo quedaría tu operación sistematizada. No es una demo limitada — es el sistema completo. No hay obligación de contratar nada después, pero la mayoría de los que lo prueban con datos reales siguen.

El onboarding es guiado — hay un wizard de 4 pasos para el admin y un tour para los alumnos nuevos. La interfaz de sesión activa (card deck con botones +/-) fue diseñada para usarse en el gimnasio, no en una oficina. El flujo de bienestar→sesión→log tarda menos de 2 minutos desde el celular. La curva de aprendizaje para el alumno es de 1 sesión.

Está en beta avanzada. El sistema tiene 85/85 tests unitarios y E2E pasando, RLS multitenant en producción con Supabase, y cobre sesiones reales con datos reales. "Beta" significa que estoy incorporando los primeros gimnasios piloto con acompañamiento directo — no significa que esté incompleto.

Si al finalizar la sesión de cierre sentís que el documento no refleja tu realidad o que las recomendaciones no son accionables, te devuelvo el 100% sin preguntas. La garantía existe porque el servicio tiene que valer lo que cuesta.
